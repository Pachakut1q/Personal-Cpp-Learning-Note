## C++面向对象特点

**三大特性:**  
1、**继承**  
让某种类型对象获得另一个类型对象的属性和方法，它可以使用现有类的所有功能，并在无需重新编写原来的类的情况下对这些功能进行扩展。  
实现继承：使用基类的属性和⽅法而无需额外编码的能力。  
接口继承：指仅使用属性和方法的名称、但是子类必须提供实现的能力。  
可视继承：子窗体（类）使用基窗体（类）的外观和实现代码的能力。  
2、**封装**  
数据和代码捆绑在⼀起，避免外界⼲扰和不确定性访问。  
把客观事物封装成抽象的类，并且类可以把⾃⼰的数据和⽅法只让可信的类或者对象操作，对不可信的进⾏信息隐
藏。  
3、**多态**  
同⼀事物表现出不同事物的能⼒，即向不同对象发送同⼀消息，不同的对象在接收时会产⽣不同的⾏为（重载实现
编译时多态，虚函数实现运⾏时多态）。  
多态性是允许你将⽗对象设置成为和⼀个或更多的他的⼦对象相等的技术，赋值之后，⽗对象就可以根据当前赋值
给它的⼦对象的特性以不同的⽅式运作。即，允许将子类类型的指针赋值给父类类型的指针。  
**实现多态有两种方式**  
覆盖（override）：子类重新定义父类的虚函数。  
重载（overload）：允许存在多个同名函数但参数表不同。
### 多态详解
C++多态性(Polymorphism)允许我们使用统一的接口来处理不同类型的对象。多态使得程序更加灵活、可扩展并易于维护。通俗地说，就是多种形态，具体点就是去完成某个行为，当不同的对象去完成时会产生不同的状态。
#### 1、构成条件
多态是在不同继承关系的类对象，去调用同一函数，产生了不同行为。  
在继承中构成多态还有两个条件：  
必须通过**基类的指针或者引用**调用虚函数；   
被调用的函数必须是虚函数，且派生类必须对基类的虚函数进行**重写**(override)。
#### 2、重写特例
1、派生类重写基类虚函数时，与基类虚函数返回值类型不同。即基类虚函数返回基类对象的指针或者引用，派生类虚函数返回派生类对象的指针或者引用。这称为**协变**。  
2、如果基类的析构函数为虚函数，此时派生类析构函数只要定义，无论是否加virtual关键字，都与基类的析构函数构成重写，虽然基类与派生类析构函数名字不同。可以理解为编译器对析构函数的名称做了特殊处理，编译后析构函数的名称统一处理成destructor。  
#### 3、C++11 override和final
final:修饰虚函数，表示该虚函数不能再被重写。
```
virtual void func() final {}
```
override:检查派生类虚函数是否重写了基类某个虚函数，如果没有重写编译报错。
```
class Base {
public:
	virtual void Func() {}
};
class Child : public Base {
public:
	virtual void Func() override { cout << "Override" << endl; }
};
```
#### 4、多态原理
**虚函数**  
有虚函数的类对象会在创建时创建一个虚函数表，虚函数表本质是一个存储虚函数指针的指针数组，一般情况下这个数组最后会放一个nullptr 。在调用虚函数时，到指定对象的虚表中去找对应的虚函数。  
**静态绑定和动态绑定**  
**静态绑定**：在编译时确定调用的函数或方法，它是通过函数或方法的名称、参数数量、类型和顺序来匹配确定的。对于非虚函数和静态成员函数，默认情况下都是静态绑定。  
**动态绑定**：是指在运行时确定调用的函数或方法，它是通过虚拟函数和指针/引用来实现的。虚拟函数是在基类中声明为虚拟的成员函数，在派生类中进行重写。通过使用基类的指针或引用调用虚拟函数时，实际调用的是派生类中重写的函数。  
#### 5、抽象类
如果在虚函数的后面写上" = 0 "，则这个函数为纯虚函数。包含纯虚函数的类叫做抽象类（接口类），抽象类不能实例化对象。派生类继承后也不能实例化出对象，只有重写纯虚函数，派生类才能实例化出对象。纯虚函数规范了派生类必须重写。
#### 6、虚表存储的位置
当一个类声明了虚函数（或继承了虚函数），编译器会隐式地在该类创建的每个对象中添加一个隐藏的指针成员，通常称为 vptr ，通常是对象内存的开头。这个 vptr 指向该类对应的 vtable 。  
虚函数表（vtable）本身被存储在程序的只读数据段，它被链接到最终的可执行文件（或共享库）中，并加载到进程地址空间的只读内存区域。同一个类的所有对象共享同一个 vtable 。



## C++11 新特性

### 智能指针
auto_ptr在 C++11 中被正式弃用，并在 C++17 中被移除。它主要有三大问题：  
复制和赋值会改变资源的所有权，不符合人的直觉；  
在STL容器中使用auto_ptr存在重大风险，因为容器内的元素必需支持可复制（copy constructable）和可赋值（assignable）；  
不支持对象数组的操作。

### 类型推导
**1、auto**  
auto可以让编译器在编译期就推导出变量的类型。  
**2、decltype**  
decltype用于推导表达式类型，只用于编译器分析表达式类型，表达式不会进行运算。
### 右值引用
**左值**：可以放在等号左边，可以取地址并有名字。  
**右值**：不可以放在等号左边，不能取地址，没有名字。  
字符串字面值是左值。  
++i、--i是**左值**， i++、i--是**右值**。  
**1、左值引用 &**  
对左值进行引用的类型，是对象的一个别名。  
**2、右值引用 &&**  
表达式等号右边的值需要是右值，可以使用std::move函数强制把左值转换为右值。  
**3、移动语义**  
可以理解为转移所有权，对于移动语义，类似于转让或者资源窃取，对于某块资源，转为自己拥有，别人不再拥有也不会再使用。  
通过移动构造函数使用移动语义，即std::move。它只针对于实现了移动构造函数的类对象。  
**浅拷贝**：a和b的指针指向了同一块内存，只是数据的简单赋值。  
**深拷贝**：在拷贝对象时，如果被拷贝对象内部还有指针引用指向其他资源，自己需要开辟新内存存储资源。  
**4、完美转发**  
写一个接受任意实参的函数模板，并转发到其他函数，目标函数会收到与转发函数完全相同的实参，通过std::forward()实现。

### nullptr
nullptr用来代替NULL。有的编译器将NULL定义为((void*)0)，有的定义为0。C++不允许直接将void*类型隐式转换到其他类型。  
如果有这样的例子：
```
void foo(char *);
void foo(int);
```
如果NULL被定义为((void*)0)，但是当编译char *ch = NULL时，NULL被定义为0；当foo(NULL)时，NULL为0会调用foo(int)，造成混乱。  
C++11引入nullptr来**区分空指针和0**。nullptr的类型为nullptr_t

### 列表初始化
采用花括号进行初始化称为列表初始化。无论是数组、对象、内置类型还是容器，列表初始化都可以用统一的语法来初始化。


### lambda表达式
lambda表达式表示一个可调用的代码单元，没有命名的内联函数，不需要函数名，一次性直接使用，其他地方不调用。  
**[capture]**(parameter list) -> return type **{function body}**  
加粗部分必需。大多数情况下，lambda表达式的返回值可由编译器猜测得出。  
**变量捕获**：  
1、[ ] 不捕获任何变量，lambda内部不能访问外部变量  
2、[&] 以引用方式捕获所有变量  
3、[=] 值的方式捕获所有变量  
4、[=, &foo] 引用捕获变量foo，其余以值方式捕获  
5、[&, foo] 值捕获foo，其余引用捕获  
6、[bar] 值捕获bar，其他变量不捕获  
7、[this] 捕获所在类的 this 指针  


## 单例模式
**单例模式**：一个类只会创建一次对象的设计模式。该类的实现只会在内存中出现一次。防止频繁地创建对象导致内存资源浪费。  
**饿汉式**：程序启动时立即初始化实例（静态初始化期）。  
优点：线程安全，无运行时检查开销。  
缺点：即使未使用也会初始化可能浪费资源，多个编译单元的静态变量初始化顺序不确定。  
**懒汉式**：首次访问时才初始化实例（延迟加载）。  
优点：按需分配，可解决初始化顺序问题。  
缺点：线程安全问题，首次访问有轻微开销。  


## 指针和引用的区别
指针是一个实体，用于存放某个对象的地址，本身就是一个变量。  
引用就是变量的别名，不可变，必须初始化。  
不存在指向空值的引用，但指针可以指向空值。  
**1、定义和声明**  
```
int x = 10;
int *ptr = &x;
```
```
int y = 20;
int &ref = y;
```
**2、使用和操作**  
**指针**：可以解引用*来访问指向的变量值，通过取地址符&获取变量地址。  
**引用**：引用在声明时被初始化，在整个生命周期中一直引用一个变量。
```
int value = *ptr; //获取变量值
int address = &x; //获取变量地址
int newValue = ref; //获取引用的值
```
**3、空值和空引用**  
指针可为空，引用不能为空。  
**4、可变性**  
指针可改变指向的不同的内存地址，引用一旦被初始化后就不能改变绑定对象。  
**5、用途**  
**指针**通常用于动态内存分配、数组操作、函数参数传递；  
**引用**通常用于函数传递参数、操作符重载、创建别名。


## i++和++i
**++i** 前缀自增，直接返回自增后的对象本身（左值引用），无临时对象。
```
int& operator++() { 
    i = i + 1;  
    return *this; // 返回当前对象的引用
}
```
**i++** 后缀自增，返回自增前的原始值副本（右值），需构造临时对象保存旧值。
```
int operator++(int) { 
    int temp = i; // 保存原始值
    i = i + 1;   // 自增
    return temp;  // 返回临时副本（非引用）
}
```
